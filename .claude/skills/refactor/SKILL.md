# Skill: Refactor
Antes de refatorar:
1. Confirmar que existe teste cobrindo o trecho
2. Aplicar impacto mínimo — mude só o necessário (mandatório pelo CLAUDE.md)
3. Não refatorar por estilo — só se for risco técnico real
4. Documentar causa + solução em `tasks/lessons.md`
5. Perguntar ao usuário se a tarefa for mal definida — não adivinhar requisitos
