# Skill: Code Review
Ao revisar código neste projeto:
1. Verificar tipagem estrita (sem `any`)
2. Checar consistência com padrões do projeto (ver CLAUDE.md)
3. Apontar problemas de performance e efeitos colaterais
4. Sugerir melhorias sem reescrever o que já funciona
5. Verificar se erros não óbvios foram registrados em `tasks/lessons.md`
